# План разработки: Химия в VR/AR — по шагам

Цель: сделать платформу по химии для школьников с разным контентом для элементов, полной таблицей и аккуратным дизайном. Задачи разбиты по шагам, чтобы не делать всё сразу и получить стабильный результат.

---

## Этап 1: Дизайн и основа сайта

### Шаг 1.1 — Дизайн-система
- [ ] Описать палитру цветов (фон, карточки, кнопки, акценты по категориям элементов).
- [ ] Выбрать шрифты: заголовки и основной текст (удобно читать школьникам).
- [ ] Задать отступы, скругления, тени (одни и те же по всему сайту).
- [ ] Создать файл с CSS-переменными (например `src/styles/variables.css`) и подключить его глобально.

### Шаг 1.2 — Главная страница
- [ ] Переработать главную: логотип/название, короткий слоган, одна главная кнопка.
- [ ] Добавить краткий блок «Как пользоваться» (2–3 пункта: таблица → элемент → 3D и VR/AR).
- [ ] Сделать адаптив: удобно на телефоне и на компьютере.
- [ ] При желании: простой фоновый паттерн или лёгкая иллюстрация (без перегруза).

### Шаг 1.3 — Общая вёрстка (layout)
- [ ] Единый шапка/подвал или только шапка с навигацией (Главная, Таблица).
- [ ] Контейнер контента с максимальной шириной и отступами.
- [ ] Одинаковый стиль кнопок и ссылок на всём сайте.

---

## Этап 2: Таблица Менделеева

### Шаг 2.1 — Внешний вид таблицы
- [ ] Улучшить сетку: размер ячеек, выравнивание номера/символа/названия.
- [ ] Подписать периоды (ряды) и группы (колонки) по желанию.
- [ ] Легенда категорий: что значит цвет (щелочные металлы, галогены и т.д.).
- [ ] При наведении: подсветка ячейки, возможно tooltip с названием.

### Шаг 2.2 — Полный набор элементов
- [ ] Добавить в данные все 118 элементов: номер, символ, название (RU/EN), row, col, category.
- [ ] Для элементов без своего текста пока использовать короткий placeholder (например: «Элемент №N — [категория]. Описание скоро.»).
- [ ] Разбить добавление на части: сначала 1–56, потом 57–88, потом 89–118, чтобы не ошибиться в row/col.

### Шаг 2.3 — Удобство использования
- [ ] Поиск по названию или символу (поле ввода + подсветка найденного в таблице).
- [ ] Фильтр по категории (показать только металлы, только галогены и т.д.).
- [ ] На мобильных: горизонтальный скролл таблицы или упрощённый вид (список элементов), если сетка не помещается.

---

## Этап 3: Контент по каждому элементу

### Шаг 3.1 — Структура страницы элемента
- [ ] Зафиксировать блоки: шапка (символ, название, номер), описание, 3D, кнопки VR/AR, при необходимости «Интересный факт» или «Где встречается».
- [ ] Сделать макет так, чтобы позже можно было добавлять разный контент (текст, картинка, видео) без переделки верстки.

### Шаг 3.2 — Уникальные описания
- [ ] Написать свои короткие описания (2–4 предложения) для 20–30 важных элементов (H, C, N, O, Na, Fe, Cu, Au и т.д.).
- [ ] Язык: простой, для школьников, без лишних формул в первом абзаце.
- [ ] Потом постепенно дополнять остальные элементы (по 10–20 за раз).

### Шаг 3.3 — Разнообразие контента по элементам
- [ ] Решить, где элемент «особенный»: своя 3D-модель (молекула, форма кристалла) или своя картинка/иконка.
- [ ] Ввести в данные поле типа контента, например: `contentType: 'atom' | 'molecule' | 'custom'` и ссылку на модель/картинку.
- [ ] Для части элементов (например вода H₂O, поваренная соль NaCl) сделать отдельные 3D-модели (молекулы), а не только общий атом.
- [ ] Остальные пока показывать через общий компонент атома (как сейчас), но с разными цветами и, при желании, разным числом орбит/электронов по данным.

---

## Этап 4: 3D и VR/AR

### Шаг 4.1 — Разные типы 3D для элементов
- [ ] Компонент «Атом» (уже есть): оставить для элементов без своей модели.
- [ ] Компонент «Молекула»: подключать для выбранных элементов (H₂O, O₂, CO₂, CH₄ и т.д.), данные — какая модель показывать.
- [ ] Единая сцена на странице элемента: по данным (contentType + modelId) рендерить либо атом, либо молекулу.
- [ ] Не перегружать страницу: одна 3D-сцена на элемент.

### Шаг 4.2 — Качество 3D
- [ ] Подобрать освещение и фон сцены так, чтобы модель было хорошо видно.
- [ ] Для атома: при желании улучшить отображение орбит/электронов (размер, прозрачность, анимация).
- [ ] Для молекул: проверить масштаб и положение в кадре (центр, не слишком далеко/близко).

### Шаг 4.3 — VR и AR
- [ ] Убедиться, что кнопки «Посмотреть в VR» и «Смотреть в AR» работают с новой логикой (одна сцена, разные типы объектов).
- [ ] Краткая подсказка на странице: на каком устройстве лучше открывать VR, на каком — AR.
- [ ] При необходимости: отдельная страница «Как пользоваться VR/AR» с пошаговой инструкцией.

---

## Этап 5: Полировка и финал

### Шаг 5.1 — Внешний вид страницы элемента
- [ ] Карточка элемента: аккуратные отступы, разделители между блоками, читаемые шрифты.
- [ ] Кнопки VR/AR: понятные подписи, контрастные, не слишком маленькие на мобильных.
- [ ] Блок с 3D: фиксированные пропорции, без «прыгающего» layout при загрузке.

### Шаг 5.2 — Навигация и ошибки
- [ ] Ссылка «Назад к таблице» на странице элемента.
- [ ] Если элемент не найден (неверный номер в URL) — понятная страница «Элемент не найден» и ссылка на таблицу.
- [ ] Проверить все ссылки и маршруты (главная → таблица → элемент → назад).

### Шаг 5.3 — Производительность и доступность
- [ ] Проверить скорость загрузки: при необходимости отложенная загрузка 3D (lazy) для страницы элемента.
- [ ] У карточек и кнопок — достаточный размер области нажатия и контраст текста/фона.
- [ ] Заголовки страниц (title) и, при возможности, описание (meta) для главной и таблицы.

---

## Порядок выполнения (рекомендация)

1. **Этап 1** — дизайн и основа (шаги 1.1 → 1.3).  
2. **Этап 2** — таблица: сначала вид и удобство (2.1), потом полный список элементов (2.2), затем поиск/фильтр (2.3).  
3. **Этап 3** — контент: структура страницы (3.1), уникальные описания (3.2), разный тип контента по элементам (3.3).  
4. **Этап 4** — 3D и VR/AR: разные типы 3D (4.1), качество (4.2), проверка VR/AR (4.3).  
5. **Этап 5** — полировка: вид страницы элемента (5.1), навигация и ошибки (5.2), производительность и доступность (5.3).

Можно выполнять по одному шагу за раз, проверять результат и только потом переходить к следующему — так проще контролировать качество и не смешивать всё в одном большом изменении.
